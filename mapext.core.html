class mapext.core.mapClass.astroMap(load_maps='IQU', **kwargs)[source]

Bases: object

Class to hold one or many astronomical maps alongside metadata describing them. Each map held inside an astroMap object is contained in a mapObj object inside the astroMap instance.

Metadata is split into overarching/common metadata (such as origin, reference, frequency) which is held in the astroMap, and map-specific metadata (such as Stokes component, projection and resolution) which is held in the mapObj.

Parameters:

  • load_maps (str, kwarg, optional) – Stokes parameters of maps to preload from supplied map details. Defaults to ‘IQU’.

  • Metadata (dict, kwarg) – Dictionary containing general metadata about the maps to be initialised. Must contain name and freq keys.

  • I (dict, kwarg, optional) – Dictionary containing metadata for the Stokes I map to be initialised. Must contain fname, unit and resolution keys, can contain header, tfield and temp keys. All keys are passed onto a mapObj instance without interpretation.

  • Q (dict, kwarg, optional) – Dictionary containing metadata for the Stokes Q map to be initialised. Must contain fname, unit and resolution keys, can contain header, tfield and temp keys. All keys are passed onto a mapObj instance without interpretation.

  • U (dict, kwarg, optional) – Dictionary containing metadata for the Stokes U map to be initialised. Must contain fname, unit and resolution keys, can contain header, tfield and temp keys. All keys are passed onto a mapObj instance without interpretation.

Raises:

ValueError – Key freq must be specified in Metadata kwarg dictionary.

Warnings:

All functions are defined for cartesian map projections, but may not be for native HEALPix map operation. Please check that your maps are suitable to be processed in cartesian before reprojecting for compatibility.

class mapext.core.mapClass.mapObj(name=None, freq=None, **kwargs)[source]

Bases: object

Class to hold a single astronomical map as part of an astroMap object. Holds map-specific metadata that can change between Stokes parameter maps from teh same source.

Parameters:

  • name (string, kwarg) – Map name. If initialised as part of astroMap object, will be set to {astromapName}_{stokesParameter}.

  • freq (string or astropy.units.Quantity, kwarg) – Map central frequency.

  • fname (str, kwarg) – Filename for data to be read from.

  • resolution (str, kwarg) – Resolution of the map as read in.

  • beamwidth (string or astropy.units.Quantity, kwarg, optional) – Beamwidth of original telescope beam before any postprocessing or smoothing. Defaults to resolution value.

  • cal (float, kwarg, optional) – Calibration uncertainty as percentage (e.g. 5% should be inputted at 5). Defaults to 8%.

  • unit (string or astropy.units.Unit, kwarg, optional) – Units of teh input map. If units are Kelvin (K), use temp keyword to specify temperature type.

  • temp (str, kwarg, optional) – Type of temperature specified in units: cmb for CMB temperature (as defined by Planck) or rj for Rayleigh-Jeans temperature.

  • zaxis (int, kwarg, optional) – index of map to read in if specified in an image cube 9greater than 2 dimensions).

class mapext.core.srcClass.astroSrc(**kwargs)[source]

Bases: object

Class to hold information pertaining to a specific astronomical source. this class is currently aimed at point sources, although will be expanded to be more flexible.

Parameters:

  • name (string, optional) – Source name

  • coords (list of astropy.coords.SkyCoord or list of floats) – Center coordinates for the source.

  • frame (string, optional) – Coordinate frame if list of floats supplied for coord.

  • flux (list of tuples) – Array of flux values (represented by tuples). Each tuple should be of the form (<name, str>, <stokes_parameter, str>, <frequency, Hz>, <resolution, arcmin>, <flux, Jy>, <std_flux, Jy>, <origin/reference, str>).

  • model (list of tuples) – Array of source models (represented by tuples). Each tuple should be of the form (<name, str>, <stokes_parameter, str>, <frequency, Hz>, <resolution, arcmin>, <model, obj>, <std_model, obj>, <origin/reference, str>).

  • flux_model (list of tuples) – Array of source models (represented by tuples). Each tuple should be of the form (<name, str>, <model, obj>, <origin/reference, str>).
